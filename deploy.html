B
Set up Supabase (your live database)
Supabase gives you a PostgreSQL database + auth + file storage — all free
~15 min
Step 1 — Create Supabase account and project
1
Go to supabase.com → click Start your project → sign in with GitHub (easiest)
2
Click New project → choose your organisation → set:
Name: nexus-asia
Database password: create a strong password (save it!)
Region: choose nearest to your users (e.g. Singapore for Asia)
3
Click Create new project. Wait ~2 minutes for it to spin up.
Step 2 — Get your API keys
1
In your Supabase project dashboard, click Settings (gear icon, left sidebar) → API
2
You'll see two important values. Copy both:
• Project URL: looks like https://abcdefghijk.supabase.co
• anon public key: a long string starting with eyJ...
⚠️ The "anon public" key is safe to include in your frontend code. It's designed to be public. Never use the "service_role" key in frontend code — it bypasses security.
Step 3 — Add your keys to config.js
1
In your GitHub repository, click on js/config.js → click the pencil (edit) icon
2
Replace lines 10–11 with your actual values:
// Replace these two lines: window.__NEXUS_SUPABASE_URL = 'https://YOUR_PROJECT_REF.supabase.co'; window.__NEXUS_SUPABASE_ANON = 'YOUR_ANON_PUBLIC_KEY_HERE'; // ↓ Becomes (example): window.__NEXUS_SUPABASE_URL = 'https://abcdefghijk.supabase.co'; window.__NEXUS_SUPABASE_ANON = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
3
Scroll down → click Commit changes. GitHub Pages will rebuild in ~60 seconds.

D
Enable Document Storage (for Doc Vault)
Supabase Storage lets you upload and store PDF reports, site photos, loan agreements
~5 min
1
In Supabase → click Storage (left sidebar) → New bucket
2
Bucket name: project-docs → keep Private (not public) → click Save
3
Run this SQL to allow authenticated users to upload/view:
CREATE POLICY "auth_upload" ON storage.objects FOR INSERT TO authenticated WITH CHECK (bucket_id = 'project-docs');
CREATE POLICY "auth_read" ON storage.objects FOR SELECT TO authenticated USING (bucket_id = 'project-docs');
✅ Document vault is now live. Team members can upload and view PDFs, Excel files, and images from any project dossier.
E
Monthly project data updates
How to keep your portfolio data current every month
Ongoing

Every month, you or your analyst team should update project data. There are 3 ways:

Option 1 — CSV Upload (recommended for teams)

Prepare a CSV with updated values for each project (DSCR, construction progress, sales velocity, escrow balance, etc). Use the Data Ingest button in the terminal to upload it. The parser will update all matching records.

Option 2 — Supabase Table Editor (for non-coders)
1
In Supabase → click Table Editor → open projects table
2
Click any row → edit the values → click the checkmark to save. Changes show in the terminal immediately on next page load.
Option 3 — SQL Update (for analysts)
-- Monthly update example (run in SQL Editor) UPDATE projects SET construction_progress = 78, dscr = 1.42, escrow_balance = 285000000, sales_velocity = 22, updated_at = NOW() WHERE id = 'PRJ001';
💡 After any update, the risk scoring engine automatically recalculates scores and alert levels on the next page load. If a project's DSCR drops below 1.0, it will automatically turn RED.
